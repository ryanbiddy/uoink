# Contributing to Uoink

Welcome! We're thrilled you want to help build Uoink. We're building the local corpus and agent layer for AI-native creators and developers.

## Getting Started

1. **Good First Issues**: If you're looking for a place to start, check our issue tracker for the `good first issue` label. These are scoped tasks perfect for your first PR.
2. **Discussions**: Have a feature idea? Let's talk about it in GitHub Discussions before you write code. This ensures your work aligns with the roadmap.
3. **Local Setup**: Clone the repo, run `npm install` for the UI, and set up your Python environment for the backend. Check the README for full instructions.

## Pull Request Process

1. **Keep it focused**: One PR should do one thing. If you're fixing a bug and adding a feature, open two PRs.
2. **Tests**: Ensure all tests pass. If you add a new feature, add a corresponding test. Run `pytest tests/` before submitting.
3. **Linting**: We enforce strict linting. Run the static checks provided in our CI workflow locally to catch issues early.

## Voice DNA (Crucial)

Any feature that generates text (prompts, UI copy, or agent outputs) must adhere to Uoink's Voice DNA. We write like sharp humans, not language models.
- **Contractions**: Use them naturally (don't, can't, won't).
- **Formatting**: Short paragraphs. 1-3 sentences max.
- **Punctuation**: NO em dashes ever. Use commas, periods, colons, or parentheses.
- **Tone**: Get to the point. No throat-clearing. Use physical verbs for abstract processes ("sanded down" not "improved").
- **Banned AI language**: Never use "delve," "harness," "landscape," "supercharge," or "game-changer." Never use "This isn't X. This is Y."

Check your copy against these rules before submitting.

Thanks for building with us!
